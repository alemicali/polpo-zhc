## RIEPILOGO COMPLETO: SISTEMA SKILLS DI POLPO

### A) Cos'è una "Skill" in Polpo
Skills sono file SKILL.md con frontmatter YAML che forniscono conoscenza
specializzata e workflow agli agenti. Vengono caricati come contesto
aggiuntivo nel system prompt al momento dello spawn dell'agente.

### B) Formato SKILL.md
```yaml
---
name: skill-name
description: Human-readable description
allowed-tools:
  - read
  - write
---
# Contenuto Markdown della skill
```

### C) Filesystem Layout
```
.polpo/skills/                  ← pool condiviso (installato con `polpo skills add`)
  frontend-design/SKILL.md
  testing/SKILL.md

.polpo/agents/                  ← assegnazione per-agente via symlink
  dev-1/skills/
    frontend-design -> ../../../skills/frontend-design
```

### D) Discovery Order
1. `<progetto>/.polpo/skills/`  — project-level pool
2. `~/.polpo/skills/`           — global skills condivisi tra progetti

### E) CLI Commands
- `polpo skills add <source>`           — Installa da GitHub o path locale
- `polpo skills list`                   — Lista skill con assegnazioni
- `polpo skills remove <name>`          — Rimuovi dal pool
- `polpo skills assign <skill> <agent>` — Assegna skill a un agente

### F) API REST Routes
- GET  /skills              — Lista skills con assegnazioni
- POST /skills/add          — Installa skills da sorgente
- DELETE /skills/:name      — Rimuovi skill
- POST /skills/:name/assign — Assegna skill a agente

### G) Sorgenti supportate per installazione
- `owner/repo`                    — GitHub shorthand
- `https://github.com/owner/repo` — Full GitHub URL
- `./local-path`                  — Path locale

   - Engine: Ink 5 (React-based terminal renderer)
   - Dipendenze: ink, ink-select-input, ink-spinner, ink-text-input, fullscreen-ink
   - Entry: `polpo tui` (comando default)
   - File: src/tui/app.tsx, store.ts, bridge.ts, format.ts, voice.ts, mentions.ts
   - Components: src/tui/components/ (Shimmer, HintBar, InputBar, TaskPanel, CompletionMenu)
   - Commands: chat, config, help, logs, memory, missions, router, sessions,
               skills, status, tasks, team, templates

2. **TUI2 (pi-tui-based)** — `src/tui2/`
   - Engine: @mariozechner/pi-tui v0.52.12
   - Entry: `polpo tui2`
   - File: src/tui2/tui.ts, bridge.ts, format.ts, theme.ts, types.ts, voice.ts, mentions.ts
   - Components: src/tui2/components/
   - Overlays: src/tui2/overlays/ (picker, viewer)
   - Commands: approvals, checkpoints, config, help, logs, memory, missions,
               modes, router, sessions, skills, status, tasks, team, templates

3. **TUI-backup (Ink, legacy)** — `dist/tui-backup/` (solo file compilati)
   - Legacy Ink-based TUI con suffisso "-ink" sui file
