## Path 1: Chat with an agent (30 seconds)

The fastest way to start. One agent, zero config, free model.

<Steps>

1. **Install**

   <Tabs>
     <Tab title="npm">
       ```bash
       npm install -g openpolpo
       ```
     </Tab>
     <Tab title="pnpm">
       ```bash
       pnpm add -g openpolpo
       ```
     </Tab>
     <Tab title="Docker">
       ```bash
       docker pull ghcr.io/openpolpo/openpolpo:latest
       docker run -it -v $(pwd):/workspace ghcr.io/openpolpo/openpolpo:latest
       ```
     </Tab>
   </Tabs>

2. **Init a project**

   ```bash
   mkdir my-project && cd my-project
   polpo init
   ```

   This creates `.polpo/polpo.json` with a default agent (`dev-1`) and the free `opencode:big-pickle` model. No API key needed.

3. **Launch**

   ```bash
   polpo
   ```

   You're in the interactive TUI. Type what you need — "create a REST API", "fix the bug in auth.ts", "write tests for the user module" — hit Enter, and the agent does it.

</Steps>

That's it. One agent, one command. If this is all you need, you're done.

## Path 2: Orchestrate a team (5 minutes)

Multiple agents, task plans with dependencies, automated assessment. This is where Polpo shines.

<Steps>

1. **Install and init**

   ```bash
   npm install -g openpolpo
   mkdir my-api && cd my-api
   polpo init
   ```

2. **Configure your team**

   Edit `.polpo/polpo.json` to define your agents:

   ```json
   {
     "project": "my-api",
     "team": {
       "name": "api-team",
       "agents": [
         {
           "name": "backend-dev",
           "role": "Backend developer specializing in Node.js, Express, and SQLite"
         },
         {
           "name": "frontend-dev",
           "role": "Frontend developer specializing in React and TypeScript"
         }
       ]
     },
     "settings": {
       "maxRetries": 3,
       "workDir": ".",
       "logLevel": "normal"
     }
   }
   ```

   <Note>
     The default model (`opencode:big-pickle`) is free and works without an API key. To use Claude, GPT-4, or Gemini instead, set `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, or `GEMINI_API_KEY` in your environment, then add `"model": "anthropic:claude-sonnet-4-5-20250929"` to any agent. See [Providers & Models](/models/overview) for 22+ supported providers.
   </Note>

3. **Create a plan**

   Use AI to generate a plan from a prompt:

   ```bash
   polpo plan create "Build a REST API with SQLite database, Express routes for users and posts, and input validation"
   ```

   Polpo generates a plan with tasks, assigns them to your agents, and sets up dependencies automatically. You can review it with:

   ```bash
   polpo plan list
   polpo plan show <plan-id>
   ```

   Or create plans interactively in the TUI with the `/plan` command.

4. **Run**

   ```bash
   polpo run
   ```

   You'll see output like:

   ```
   [14:00:01] Polpo started — my-api
   [14:00:01] Team agents: backend-dev, frontend-dev

   [14:00:02] [task-1] Task added: Create database layer
   [14:00:02] [task-2] Task added: Build REST endpoints
   [14:00:02] [task-1] Spawning "backend-dev" for: Create database layer
   [14:24:15] [task-1] PASSED (score: 4.2/5)
   [14:24:15] [task-1] DONE — Create database layer
   [14:24:17] [task-2] Spawning "backend-dev" for: Build REST endpoints
   [14:25:30] [task-2] PASSED (score: 4.0/5)
   [14:25:30] [task-2] DONE — Build REST endpoints

       All 2 tasks completed! (1m29s)
   ```

5. **Monitor** (optional)

   In a second terminal:

   ```bash
   polpo status -w
   ```

   Or start the HTTP server for the Web UI and REST API:

   ```bash
   polpo serve
   # API at http://localhost:3000/api/v1/
   ```

</Steps>

## What just happened?

1. **Plan** — Polpo's AI generated a plan with tasks, dependencies, and agent assignments
2. **Assign** — The orchestrator assigned each task to the right agent, respecting dependencies
3. **Execute** — Detached runner processes spawned agents with coding tools (read, write, edit, bash, glob, grep, ls)
4. **Assess** — Three LLM judges scored each result on correctness, completeness, code quality, and edge cases
5. **Done** — Tasks that pass move to `done`. Tasks that fail get feedback and retry automatically
