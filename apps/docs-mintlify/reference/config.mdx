## Top-Level Structure

```json
{
  "version": "1",
  "project": "my-project",
  "team": {
    "name": "My Team",
    "agents": []
  },
  "tasks": [],
  "settings": {},
  "providers": {}
}
```

## Agent Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | string | — | **Required.** Unique agent identifier |
| `role` | string | — | Agent's role description |
| `model` | string | — | LLM model. Format: `provider:model` (e.g. `"anthropic:claude-sonnet-4-5-20250929"`) or bare model ID |
| `allowedTools` | string[] | — | Allowed tool names |
| `allowedPaths` | string[] | `[workDir]` | Filesystem sandbox — directories the agent can access. Absolute or relative to `workDir` |
| `mcpServers` | object | — | MCP server configuration. Keys are server names, values are stdio or HTTP configs |
| `systemPrompt` | string | — | Additional system prompt appended to base |
| `skills` | string[] | — | Installed skill names (e.g. `"frontend-design"`) |
| `maxTurns` | number | `150` | Max conversation turns before stopping |
| `maxConcurrency` | number | unlimited | Max concurrent tasks for this agent |
| `volatile` | boolean | `false` | Auto-cleanup when plan completes |
| `planGroup` | string | — | Plan group this volatile agent belongs to |
| `enableBrowser` | boolean | `false` | Enable browser automation tools (18 tools, requires `agent-browser`) |
| `enableHttp` | boolean | `false` | Enable HTTP fetch/download tools |
| `enableGit` | boolean | `false` | Enable structured git tools |
| `enableMultifile` | boolean | `false` | Enable batch file editing tools |
| `enableDeps` | boolean | `false` | Enable dependency management tools (npm/pnpm/yarn/bun) |
| `enableExcel` | boolean | `false` | Enable Excel/CSV tools |
| `enablePdf` | boolean | `false` | Enable PDF tools |
| `enableDocx` | boolean | `false` | Enable Word/DOCX tools |
| `enableEmail` | boolean | `false` | Enable email tools (requires SMTP env vars) |
| `enableAudio` | boolean | `false` | Enable speech-to-text and text-to-speech tools |
| `enableImage` | boolean | `false` | Enable AI image generation and vision analysis tools |

## Task Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | string | auto | Auto-generated unique ID |
| `title` | string | — | **Required.** Short task title |
| `description` | string | — | **Required.** Detailed instructions |
| `assignTo` | string | — | **Required.** Agent name |
| `group` | string | — | Plan group name |
| `dependsOn` | string[] | `[]` | Task titles this depends on |
| `status` | TaskStatus | `"pending"` | Current status |
| `expectations` | Expectation[] | `[]` | Assessment criteria |
| `metrics` | Metric[] | `[]` | Custom metrics |
| `maxRetries` | number | `3` | Max full retry attempts |
| `maxDuration` | number | `1800000` | Timeout in ms (0 = no timeout) |
| `retryPolicy` | RetryPolicy | — | Escalation settings |

## TaskExpectation Fields

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | **Required.** `"test"`, `"file_exists"`, `"script"`, or `"llm_review"` |
| `command` | string | Shell command (for `test` and `script`) |
| `paths` | string[] | File paths to check (for `file_exists`) |
| `criteria` | string | Review prompt (for `llm_review`) |
| `dimensions` | EvalDimension[] | Scoring dimensions (for `llm_review`) |
| `threshold` | number | Minimum passing score 1-5 (default: `3.0`) |

## EvalDimension Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Dimension name (e.g. `"correctness"`) |
| `description` | string | What this dimension measures |
| `weight` | number | Weight 0-1 (should sum to ~1 across dimensions) |
| `rubric` | object | Score descriptions per level (1-5) |

## RetryPolicy Fields

| Field | Type | Description |
|-------|------|-------------|
| `escalateAfter` | number | Failure count before escalating |
| `fallbackAgent` | string | Agent for escalation retries |
| `escalateModel` | string | Model override for escalation |

## Settings Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `maxRetries` | number | `3` | Default max retries per task |
| `workDir` | string | `"."` | Working directory |
| `logLevel` | string | `"normal"` | `"quiet"`, `"normal"`, or `"verbose"` |
| `taskTimeout` | number | `1800000` | Default task timeout (ms) |
| `staleThreshold` | number | `300000` | Idle time before stale warning (ms) |
| `storage` | string | `"file"` | Storage backend: `"file"` (filesystem JSON) or `"sqlite"` |
| `maxConcurrency` | number | unlimited | Max concurrent agent processes globally |
| `maxAssessmentRetries` | number | `1` | Max assessment retries when all reviewers fail before falling back to fix/retry |
| `autoCorrectExpectations` | boolean | `true` | Auto-correct correctable expectations (e.g. file_exists paths) on assessment failure |
| `defaultRetryPolicy` | RetryPolicy | — | Default retry policy for all tasks |
| `enableVolatileTeams` | boolean | `true` | Allow volatile agents in plan `team:` sections |
| `volatileCleanup` | string | `"on_complete"` | `"on_complete"` or `"manual"` |
| `maxFixAttempts` | number | `2` | Max fix attempts before full retry. During the fix phase, agents receive targeted feedback to correct specific issues instead of starting over. After `maxFixAttempts` fail, the task falls back to a full retry. See [Escalation Chain](/features/escalation). |
| `maxQuestionRounds` | number | `2` | Max auto-answer rounds per task. When an agent asks a question, Polpo generates an answer using project context. After `maxQuestionRounds`, Q&A stops to prevent infinite loops. See [Question Detection](/features/question-detection). |
| `maxResolutionAttempts` | number | `2` | Max deadlock resolution attempts per blocked task. When a task depends on a failed dependency, Polpo evaluates whether to absorb the failure, retry the dependency, or fail the task. Each blocked task gets at most `maxResolutionAttempts` before being permanently failed. See [Deadlock Resolution](/features/deadlock-resolution). |
| `orchestratorModel` | string | — | Model for orchestrator LLM calls |
| `approvalGates` | ApprovalGate[] | — | Approval gate checkpoints. See [Approval Gates](/features/approval-gates) |
| `notifications` | object | — | Notification routing configuration. See [Notifications](/features/notifications) |
| `escalationPolicy` | object | — | Default escalation chain. See [Escalation](/features/escalation) |
| `sla` | object | — | SLA monitoring configuration. See [Scoring](/features/scoring) |
| `enableScheduler` | boolean | auto | Enable the scheduling engine. Default: `true` if any plan has a schedule |
| `defaultQualityThreshold` | number | — | Default quality threshold for plans (1-5). Plans below this score are marked failed |

## ProjectConfig Fields

Stored in `.polpo/config.json` — separate from `polpo.json`, this stores project-level defaults:

| Field | Type | Description |
|-------|------|-------------|
| `project` | string | Project name |
| `judge` | string | Model for assessment LLM |
| `model` | string | Default model |

## Providers

Per-provider API key and base URL overrides. Keys are provider names, values can be a string (shorthand for API key) or an object with `apiKey` and `baseUrl`.

```json
{
  "providers": {
    "anthropic": "${ANTHROPIC_API_KEY}",
    "openai": {
      "apiKey": "${OPENAI_API_KEY}"
    },
    "openrouter": {
      "apiKey": "sk-or-...",
      "baseUrl": "https://openrouter.ai/api/v1"
    }
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `apiKey` | string | API key. Supports `${ENV_VAR}` references resolved at startup |
| `baseUrl` | string | Override base URL (for custom proxies or self-hosted endpoints) |

See [Providers & Models](/models/overview) for supported providers and model resolution.

## Environment Variables

### Polpo-specific

| Variable | Description |
|----------|-------------|
| `POLPO_MODEL` | Global default model override (e.g. `anthropic:claude-sonnet-4-5-20250929`) |
| `POLPO_JUDGE_MODEL` | Model for LLM-as-Judge assessment reviews |
| `POLPO_STT_MODEL` | Speech-to-text model (default: `gpt-4o-mini-transcribe`) |
| `API_KEY` | Server authentication key (for HTTP API) |

### Provider API Keys

| Variable | Provider |
|----------|----------|
| `ANTHROPIC_API_KEY` | Anthropic (Claude) |
| `OPENAI_API_KEY` | OpenAI (GPT, o-series, DALL-E, Whisper, TTS) |
| `GEMINI_API_KEY` | Google (Gemini) |
| `GROQ_API_KEY` | Groq (Llama) |
| `MISTRAL_API_KEY` | Mistral |
| `XAI_API_KEY` | xAI (Grok) |
| `OPENROUTER_API_KEY` | OpenRouter |
| `CEREBRAS_API_KEY` | Cerebras |
| `HF_TOKEN` | Hugging Face |
| `OPENCODE_API_KEY` | OpenCode (free default) |
| `DEEPGRAM_API_KEY` | Deepgram (audio transcription) |
| `ELEVENLABS_API_KEY` | ElevenLabs (text-to-speech) |
| `REPLICATE_API_TOKEN` | Replicate (image generation) |

### Extended Tool Environment Variables

| Variable | Description |
|----------|-------------|
| `SMTP_HOST` | SMTP host for email tools |
| `SMTP_USER` | SMTP username |
| `SMTP_PASS` | SMTP password |

## Full Example

```json
{
  "providers": {
    "anthropic": "${ANTHROPIC_API_KEY}",
    "openai": "${OPENAI_API_KEY}"
  },
  "agents": [
    {
      "name": "architect",
      "role": "Senior architect",
      "model": "anthropic:claude-sonnet-4-5-20250929",
      "maxTurns": 200,
      "systemPrompt": "Always consider performance implications"
    },
    {
      "name": "developer",
      "role": "Full-stack developer",
      "skills": ["frontend-design"]
    },
    {
      "name": "script-runner",
      "role": "Script executor"
    }
  ],
  "plans": [
    {
      "group": "build-feature",
      "team": [
        {
          "name": "specialist",
          "role": "Feature specialist",
          "volatile": true
        }
      ],
      "tasks": [
        {
          "title": "Design database schema",
          "agent": "architect",
          "description": "Design normalized schema for user management"
        },
        {
          "title": "Implement API",
          "agent": "developer",
          "description": "Build Express REST API",
          "dependencies": ["Design database schema"],
          "maxDuration": 3600000,
          "retryPolicy": {
            "escalateAfter": 2,
            "fallbackAgent": "architect"
          },
          "expectations": [
            { "type": "file_exists", "paths": ["src/routes.ts", "src/db.ts"] },
            { "type": "test", "command": "npm test" },
            {
              "type": "llm_review",
              "criteria": "Verify security and error handling",
              "threshold": 4.0,
              "dimensions": [
                { "name": "security", "weight": 0.6, "description": "No vulnerabilities" },
                { "name": "error_handling", "weight": 0.4, "description": "Proper error responses" }
              ]
            }
          ]
        }
      ]
    }
  ],
  "settings": {
    "maxRetries": 3,
    "workDir": ".",
    "logLevel": "normal",
    "storage": "file",
    "taskTimeout": 1800000,
    "staleThreshold": 300000,
    "maxConcurrency": 4,
    "maxFixAttempts": 2,
    "maxQuestionRounds": 2,
    "autoCorrectExpectations": true,
    "defaultQualityThreshold": 3.0,
    "orchestratorModel": "claude-sonnet-4-5-20250929"
  }
}
```
