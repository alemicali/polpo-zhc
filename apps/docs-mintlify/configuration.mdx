Polpo is configured through a JSON file at `.polpo/polpo.json` in your project directory. Run `polpo init` to create one with defaults.

## File Structure

```json
{
  "project": "my-project",
  "team": {
    "name": "default",
    "description": "Project team description",
    "agents": [
      {
        "name": "agent-name",
        "role": "Role description",
        "model": "opencode:big-pickle"
      }
    ]
  },
  "providers": {
    "anthropic": "${ANTHROPIC_API_KEY}",
    "openai": "${OPENAI_API_KEY}"
  },
  "settings": {
    "maxRetries": 3,
    "workDir": ".",
    "logLevel": "normal"
  }
}
```

<Note>
Plans and tasks are managed dynamically via the TUI, CLI, or API. They are stored separately in `.polpo/plans/` and `.polpo/tasks/` — **not** in the config file.
</Note>

## Agents

Each agent entry defines a worker that can be assigned tasks. You specify a `name`, a `role` describing its expertise, and optionally a `model` for the LLM to use.

```json
{
  "team": {
    "agents": [
      {
        "name": "developer",
        "model": "opencode:big-pickle",
        "role": "Full-stack developer"
      },
      {
        "name": "architect",
        "model": "anthropic:claude-sonnet-4-5-20250929",
        "role": "Senior software architect who designs system components"
      }
    ]
  }
}
```

Agents support many additional options like `allowedTools`, `allowedPaths`, `enableBrowser`, `skills`, and more. See [Configuration Reference](/reference/config) for the full field list.

## Creating Plans

Plans are created via the CLI or TUI — not by editing the config file:

<Tabs>
  <Tab title="CLI">
    ```bash
    # AI-generated plan from a prompt
    polpo plan create "Build a REST API with auth and tests"

    # List and inspect plans
    polpo plan list
    polpo plan show <plan-id>

    # Execute a plan
    polpo plan execute <plan-id>
    ```
  </Tab>
  <Tab title="TUI">
    ```
    /plan
    ```
    The TUI plan command lets you create plans from templates or generate them with AI.
  </Tab>
  <Tab title="HTTP API">
    ```bash
    # Create a plan
    curl -X POST http://localhost:3000/api/v1/projects/my-project/plans \
      -H "Content-Type: application/json" \
      -d '{"prompt": "Build a REST API with auth"}'

    # Execute a plan
    curl -X POST http://localhost:3000/api/v1/projects/my-project/plans/<plan-id>/execute
    ```
  </Tab>
</Tabs>

Each plan contains tasks with these key fields:

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Short task title |
| `description` | string | Detailed instructions |
| `assignTo` | string | Agent name from your team |
| `dependsOn` | string[] | Titles of tasks that must finish first |
| `expectations` | array | Assessment criteria |

Plans can also define volatile teams (agents scoped to a single plan) and quality gates. See [Plans & Dependencies](/concepts/plans) for details on plan structure.

## Settings

The `settings` object controls Polpo's behavior — retry limits, tick interval, log level, stale detection thresholds, and more:

```json
{
  "settings": {
    "maxRetries": 3,
    "maxFixAttempts": 2,
    "staleThreshold": 300000,
    "logLevel": "normal",
    "workDir": ".",
    "orchestratorModel": "claude-sonnet-4-5-20250929"
  }
}
```

## Optional Systems

Polpo has several optional capabilities that can be enabled in the config:

| System | Config Key | Learn More |
|--------|------------|------------|
| Approval gates | `approvalGates` | [Approval Gates](/features/approval-gates) |
| Notifications | `notifications` | [Notifications](/features/notifications) |
| Escalation | `escalationPolicy` | [Escalation Chain](/features/escalation) |
| SLA monitoring | `sla` | [Scoring](/features/scoring) |
| Scheduling | `enableScheduler` | [Scheduling](/features/scheduling) |
| Quality threshold | `defaultQualityThreshold` | [Scoring](/features/scoring) |

## Environment Variables

```bash
# Optional: use a specific provider
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...

# Optional: customize orchestrator LLM
POLPO_MODEL=anthropic:claude-sonnet-4-5-20250929
POLPO_JUDGE_MODEL=anthropic:claude-sonnet-4-5-20250929

# Optional: server authentication
API_KEY=your-secret-key
```

## Config Path

By default, Polpo looks for `.polpo/polpo.json` in the current directory. Use `-d` or `--dir` to specify a different working directory:

```bash
polpo run -d /path/to/workdir
polpo -d /path/to/workdir
```
