---

This guide walks you through creating a multi-task plan, executing it, and monitoring the results.

## What We'll Build

A simple API project with two tasks:
1. A backend developer creates the database layer
2. The same developer builds REST endpoints on top of it

<Steps>

1. **Initialize Polpo**

   ```bash
   mkdir my-api-project && cd my-api-project
   polpo init
   ```

2. **Set your API key** (optional)

   The default model (`opencode:big-pickle`) is free and works without any key. To use Claude instead:

   ```bash
   export ANTHROPIC_API_KEY=sk-ant-...
   ```

3. **Configure your team**

   Edit `.polpo/polpo.json`:

   ```json
   {
     "project": "my-api-project",
     "team": {
       "name": "api-team",
       "agents": [
         {
           "name": "api-dev",
           "role": "Backend developer specializing in Node.js, Express, and SQLite. Writes clean, well-tested code with proper error handling."
         }
       ]
     },
     "settings": {
       "maxRetries": 3,
       "workDir": ".",
       "logLevel": "normal"
     }
   }
   ```

   <Note>
   Plans and tasks are **not** defined in `polpo.json`. The config file only contains team and settings. Plans are created via the CLI or TUI and stored in `.polpo/plans/`.
   </Note>

4. **Create the plan**

   Use AI to generate a plan:

   ```bash
   polpo plan create "Build a blog API: first create a SQLite database layer with users and posts tables, then build Express REST endpoints on top of it. The API task depends on the database task."
   ```

   This creates a plan in `.polpo/plans/` with tasks, dependencies, and agent assignments. Review it:

   ```bash
   polpo plan list
   polpo plan show <plan-id>
   ```

   The plan will contain tasks like:

   ```
   Task 1: Create database layer
     assignTo: api-dev
     dependsOn: []

   Task 2: Build REST endpoints
     assignTo: api-dev
     dependsOn: ["Create database layer"]
   ```

   <Tip>
   You can also create plans interactively in the TUI using the `/plan` command, which offers AI generation and manual editing.
   </Tip>

5. **Run the plan**

   ```bash
   polpo run
   ```

   You'll see output like:

   ```
   [14:00:01] Polpo started — my-api-project
   [14:00:01] Team agents: api-dev

   [14:00:02] [task-1] Task added: Create database layer
   [14:00:02] [task-2] Task added: Build REST endpoints
   [14:00:02] [task-1] Spawning "api-dev" for: Create database layer
   ```

6. **Monitor progress**

   Open a second terminal and watch:

   ```bash
   polpo status -w
   ```

   This shows a live dashboard with task status, agent activity, and scores.

7. **Review results**

   When complete, you'll see:

   ```
   [14:02:30] [task-1] PASSED (score: 4.2/5)
   [14:02:30] [task-1] DONE — Create database layer
   [14:02:32] [task-2] Spawning "api-dev" for: Build REST endpoints
   [14:03:45] [task-2] PASSED (score: 4.0/5)
   [14:03:45] [task-2] DONE — Build REST endpoints

       All 2 tasks completed! (3m44s)
   ```

</Steps>

## Understanding the Output

### Assessment Scores

Each task gets scored on 4 dimensions (1-5):

| Dimension | Weight | What It Measures |
|-----------|--------|-----------------|
| Correctness | 35% | Does the code work correctly? |
| Completeness | 30% | Are all requirements addressed? |
| Code Quality | 20% | Is the code clean and maintainable? |
| Edge Cases | 15% | Are error conditions handled? |

A task **passes** when the weighted average score is >= 3.0 (configurable).

### What If a Task Fails?

If assessment fails, Polpo automatically:
1. Sends detailed feedback to the agent (per-dimension scores + reasoning)
2. Enters **fix phase** — agent makes targeted corrections
3. Re-assesses up to `maxFixAttempts` (default: 2) times
4. Falls back to **full retry** if fixes aren't enough
5. Gives up after `maxRetries` (default: 3) total attempts

## Plan Task Fields Reference

When plans are generated by AI or created via the API, each task uses these fields:

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Short task title (used for dependency references) |
| `description` | string | Detailed instructions for the agent |
| `assignTo` | string | Agent name from your team config |
| `dependsOn` | string[] | Titles of tasks that must finish first |
| `expectations` | array | Assessment criteria (optional) |
| `maxRetries` | number | Override default retry limit (optional) |
