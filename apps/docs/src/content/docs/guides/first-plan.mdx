This guide walks you through creating a multi-task plan, executing it, and monitoring the results.

## What We'll Build

A simple API project with two tasks:
1. A backend developer creates the database layer
2. The same developer builds REST endpoints on top of it

<Steps>

1. **Initialize Polpo**

   ```bash
   mkdir my-api-project && cd my-api-project
   polpo init
   ```

2. **Set your API key**

   ```bash
   export ANTHROPIC_API_KEY=sk-ant-...
   ```

3. **Configure your team**

   The `.polpo/polpo.json` file contains your team configuration. Plans are created interactively via TUI or CLI, not in the config file.

   ```yaml
   agents:
     - name: api-dev
       adapter: claude-sdk
       description: >
         Backend developer specializing in Node.js, Express, and SQLite.
         Writes clean, well-tested code with proper error handling.

   plans:
     - group: build-api
       tasks:
         - title: Create database layer
           agent: api-dev
           description: |
             Create a SQLite database layer for a blog application:
             1. Install better-sqlite3
             2. Create src/db.ts with:
                - users table (id, name, email, created_at)
                - posts table (id, title, body, author_id, created_at)
             3. Create a seed script that adds sample data
             4. Export typed query functions: getUsers, getUser, createUser, getPosts, createPost

           expectations:
             - type: file_exists
               paths:
                 - src/db.ts
             - type: test
               command: "node -e \"require('./src/db')\""

         - title: Build REST endpoints
           agent: api-dev
           description: |
             Build Express REST API endpoints using the database layer from src/db.ts:
             - GET /users - list all users
             - GET /users/:id - get single user
             - POST /users - create user (validate email)
             - GET /posts - list all posts (with author info)
             - POST /posts - create post (require title and body)
             Add proper error handling and input validation.
           dependencies:
             - Create database layer
           expectations:
             - type: file_exists
               paths:
                 - src/routes.ts
             - type: llm_review
               prompt: Verify proper input validation, error handling, and REST conventions
               threshold: 3.5
   ```

4. **Run the plan**

   ```bash
   polpo run
   ```

   You'll see output like:

   ```
   [14:00:01] Polpo started — my-api-project
   [14:00:01] Team agents: api-dev

   [14:00:02] [task-1] Task added: Create database layer
   [14:00:02] [task-2] Task added: Build REST endpoints
   [14:00:02] [task-1] Spawning "api-dev" for: Create database layer
   ```

5. **Monitor progress**

   Open a second terminal and watch:

   ```bash
   polpo status -w
   ```

   This shows a live dashboard with progress bars, agent activity, and scores.

6. **Review results**

   When complete, you'll see:

   ```
   [14:02:30] [task-1] PASSED (score: 4.2/5)
   [14:02:30] [task-1] DONE — Create database layer
   [14:02:32] [task-2] Spawning "api-dev" for: Build REST endpoints
   [14:03:45] [task-2] PASSED (score: 4.0/5)
   [14:03:45] [task-2] DONE — Build REST endpoints

       All 2 tasks completed! (3m44s)
   ```

</Steps>

## Understanding the Output

### Assessment Scores

Each task gets scored on 4 dimensions (1-5):

| Dimension | Weight | What It Measures |
|-----------|--------|-----------------|
| Correctness | 35% | Does the code work correctly? |
| Completeness | 30% | Are all requirements addressed? |
| Code Quality | 20% | Is the code clean and maintainable? |
| Edge Cases | 15% | Are error conditions handled? |

A task **passes** when the weighted average score is >= 3.0 (configurable).

### What If a Task Fails?

If assessment fails, Polpo automatically:
1. Sends detailed feedback to the agent (per-dimension scores + reasoning)
2. Enters **fix phase** — agent makes targeted corrections
3. Re-assesses up to `maxFixAttempts` (default: 2) times
4. Falls back to **full retry** if fixes aren't enough
5. Gives up after `maxRetries` (default: 3) total attempts

## Next Steps

- **Add more agents**: Have different agents work on different aspects
- **Custom expectations**: Add test commands and specific file checks
- **Dependencies**: Build complex task graphs with `dependencies`
- **Volatile teams**: Create temporary specialists per plan

<Aside>
For interactive monitoring, use `polpo tui` instead of `polpo run`. The TUI provides real-time dashboards, chat, and plan management.
