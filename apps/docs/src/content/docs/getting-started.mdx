## Prerequisites

- **Node.js** v18 or higher
- **npm** v8 or higher
- **API Keys** (optional): Only required for paid models. The default `opencode:big-pickle` model is free and requires no API key.

## Installation

<Steps>

1. **Clone the repository**

   ```bash
   git clone https://github.com/openpolpo/openpolpo.git
   cd openpolpo
   ```

2. **Install dependencies**

   ```bash
   npm install
   ```

3. **Build the project**

   ```bash
   pnpm run build
   ```

4. **Link the CLI globally** (optional)

   ```bash
   npm link
   ```

</Steps>

## Configure Your Environment (Optional)

The default `opencode:big-pickle` model works out-of-the-box without any API key.

If you want to use a paid model, set the appropriate API key:

```bash
# For Anthropic Claude models
export ANTHROPIC_API_KEY=sk-ant-...

# For OpenAI models
export OPENAI_API_KEY=sk-...

# For other providers
export GOOGLE_API_KEY=...
export GROQ_API_KEY=gsk_...
```

## Initialize a Project

```bash
polpo init
```

This creates a `.polpo/` directory with `polpo.json` configuration file containing your project setup and default agents.

## Write Your First Plan

The `.polpo/polpo.json` file contains your team configuration. Plans are created dynamically via the TUI or CLI.

Your initial config looks like:

```json
{
  "project": "my-project",
  "team": {
    "name": "default",
    "agents": [
      {
        "name": "backend-dev",
        "model": "opencode:big-pickle",
        "role": "Backend developer specializing in Node.js and databases"
      },
      {
        "name": "frontend-dev",
        "model": "opencode:big-pickle",
        "role": "Frontend developer specializing in React and TypeScript"
      }
    ]
  },
  "plans": [
    {
      "group": "setup-project",
      "tasks": [
        {
          "title": "Create database schema",
          "agent": "backend-dev",
          "description": "Design and implement a SQLite database schema for users and posts. Create migration files and a seed script."
        },
        {
          "title": "Build React UI components",
          "agent": "frontend-dev",
          "description": "Create reusable UI components using shadcn/ui: UserCard, PostList, CreatePostForm"
        }
      ]
    }
  ]
}
```

## Run Your Plan

<Tabs>
  <TabItem label="TUI (Interactive)">
    ```bash
    polpo
    ```
    The TUI provides real-time monitoring with tabs for Dashboard, Tasks, Plans, Agents, Logs, and Chat.
  </TabItem>

  <TabItem label="Headless">
    ```bash
    polpo run
    ```
    Runs all pending plans without interactive UI. Logs output to stdout.
  </TabItem>

  <TabItem label="Server">
    ```bash
    polpo serve --port 3890
    ```
    Starts an HTTP API server with SSE streaming and WebSocket support.
  </TabItem>
</Tabs>

## What Happens Next

When you run a plan, Polpo:

1. **Assigns** each task to the specified agent
2. **Spawns** detached runner processes for each agent
3. **Monitors** agent progress with periodic health checks
4. **Assesses** completed work using G-Eval (LLM-as-judge)
5. **Retries** tasks that don't meet the quality threshold
6. **Cleans up** volatile agents when their plan completes
