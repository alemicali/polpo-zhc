Polpo is configured through a JSON file at `.polpo/polpo.json` in your project directory.

## File Structure

```json
{
  "project": "my-project",
  "team": {
    "name": "default",
    "description": "Project team description",
    "agents": [
      {
        "name": "agent-name",
        "adapter": "native",
        "role": "Role description",
        "model": "opencode:big-pickle"
      }
    ]
  },
  "providers": {
    "anthropic": "${ANTHROPIC_API_KEY}",
    "openai": "${OPENAI_API_KEY}"
  },
  "settings": {
    "maxRetries": 3,
    "workDir": ".",
    "logLevel": "normal"
  }
}
```

<Aside type="note">
Plans and tasks are managed dynamically via the TUI/CLI/API and stored in `.polpo/plans/*.json` and task stores — they are NOT defined in the config file.
</Aside>

## Agents

Each agent entry defines a worker that can be assigned tasks:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | yes | Unique identifier for the agent |
| `adapter` | string | no | `"native"` (default), `"claude-sdk"`, or `"generic"` |
| `role` | string | no | Agent's role and expertise description |
| `model` | string | native only | LLM model (e.g., `opencode:big-pickle`, `anthropic:claude-sonnet-4-5`) |
| `command` | string | generic only | CLI command to execute |
| `volatile` | boolean | no | Auto-cleanup when plan completes (default: `false`) |

```json
{
  "team": {
    "agents": [
      {
        "name": "developer",
        "adapter": "native",
        "model": "opencode:big-pickle",
        "role": "Full-stack developer using Polpo's native engine"
      },
      {
        "name": "architect",
        "adapter": "claude-sdk",
        "model": "claude-sonnet-4-5-20250929",
        "role": "Senior software architect who designs system components"
      },
      {
        "name": "aider-dev",
        "adapter": "generic",
        "command": "aider --no-auto-commit --model gpt-4",
        "role": "Aider-powered developer for code changes"
      }
    ]
  }
}
```

## Plans

Plans group related tasks into execution units:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `group` | string | yes | Unique plan identifier |
| `team` | Agent[] | no | Volatile agents scoped to this plan |
| `tasks` | Task[] | yes | List of tasks to execute |

### Volatile Teams

Agents defined in the `team` section are automatically created when the plan starts and cleaned up when it completes:

```yaml
plans:
  - group: feature-auth
    team:
      - name: auth-specialist
        adapter: claude-sdk
        description: Authentication expert
        volatile: true
    tasks:
      - title: Implement JWT auth
        agent: auth-specialist
        description: Build JWT authentication middleware
```

## Tasks

Each task is a unit of work assigned to an agent:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | yes | Short task title |
| `agent` | string | yes | Agent name to assign to |
| `description` | string | yes | Detailed instructions for the agent |
| `dependencies` | string[] | no | Task IDs that must complete first |
| `expectations` | Expectation[] | no | Custom assessment criteria |

## Expectations

Define how completed tasks are assessed:

```yaml
tasks:
  - title: Build API endpoints
    agent: backend-dev
    description: Create REST API for user management
    expectations:
      - type: test
        command: npm test
        weight: 0.4

      - type: file_exists
        path: src/routes/users.ts
        weight: 0.2

      - type: llm_review
        criteria: Check for proper error handling and input validation
        weight: 0.4
        threshold: 3.5
        dimensions:
          - name: security
            weight: 0.5
            rubric: No SQL injection or XSS vulnerabilities
          - name: error_handling
            weight: 0.5
            rubric: All error paths return proper HTTP status codes
```

See [Assessment](/concepts/assessment/) for details on the scoring system.

## Environment Variables

```bash
# Required for Claude agents
ANTHROPIC_API_KEY=sk-ant-...

# Optional: customize orchestrator LLM
ORCHESTRATOR_MODEL=claude-sonnet-4-5-20250929
ORCHESTRATOR_TEMPERATURE=0.7

# Optional: server authentication
API_KEY=your-secret-key
```

## Config Path

By default, Polpo looks for `.polpo/polpo.json` in the current directory. Use `-d` or `--dir` to specify a different working directory:

```bash
polpo run -d /path/to/workdir
polpo tui -d /path/to/workdir
```

The config file must always be at `.polpo/polpo.json` relative to the working directory.
