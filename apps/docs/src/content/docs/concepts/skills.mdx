Skills are **markdown instruction files** that give an agent specialized knowledge. They're like mini system prompts that you can mix and match across agents — encoding team conventions, coding standards, or domain-specific knowledge without bloating every agent's config.

## What's a Skill?

A skill is a directory containing a `SKILL.md` file:

```
.polpo/skills/typescript-patterns/
  SKILL.md
```

The `SKILL.md` file has YAML frontmatter and a markdown body:

```markdown
---
description: TypeScript best practices and common patterns
allowedTools:
  - read
  - write
  - edit
---

## TypeScript Patterns

When writing TypeScript code, follow these conventions:

- Use `const` by default, `let` only when reassignment is needed
- Prefer interfaces over type aliases for object shapes
- Use discriminated unions for state machines
...
```

The frontmatter `description` helps Polpo tell agents what a skill does without loading its full content. The optional `allowedTools` field informs which tools the agent should have when this skill is active.

## Skill Sources

Polpo discovers skills from three locations (in order of priority):

| Location | Scope | Description |
|----------|-------|-------------|
| `.polpo/skills/` | Project | Skills shared across all agents in this project |
| `.polpo/agents/<name>/skills/` | Agent | Skills assigned to a specific agent (via symlinks) |
| `~/.claude/skills/` | User | User-level skills available in all projects |

## Assigning Skills

You can assign skills to agents in two ways:

**In config** — list skill names in the agent's `skills` array:

```json
{
  "name": "frontend-dev",
  "skills": ["react-patterns", "css-conventions"]
}
```

**Via symlinks** — create a symlink in the agent's skill directory:

```bash
# Assign the "react-patterns" skill to "frontend-dev"
ln -s ../../skills/react-patterns .polpo/agents/frontend-dev/skills/react-patterns
```

## How Skills Are Loaded

When an agent starts a task:

1. Polpo scans all skill sources and discovers available skills
2. Skills matching the agent's `skills` config (or symlinked) are loaded
3. The skill markdown content is injected into the agent's system prompt
4. Skills with `allowedTools` also inform which tools the agent receives

Skills that aren't loaded for a particular agent are still mentioned by name in a "you have not loaded these skills" section — so the agent knows they exist but doesn't get their content.

## Creating a Skill

```bash
mkdir -p .polpo/skills/my-skill
```

Then create `.polpo/skills/my-skill/SKILL.md` with your instructions. That's it — Polpo will discover it automatically on the next agent run.

<Aside type="tip">
Skills are a great way to keep agent instructions DRY. Instead of copy-pasting the same coding standards into every agent's `systemPrompt`, create a skill once and assign it to multiple agents.
</Aside>

## Examples

### Code Review Skill

```markdown
---
description: Code review standards and quality criteria
allowedTools:
  - read
  - glob
  - grep
---

## Code Review

When reviewing code, check for:

1. **Security** — no hardcoded secrets, proper input validation, SQL injection prevention
2. **Error handling** — all async operations have try/catch, errors are logged with context
3. **Performance** — no N+1 queries, efficient data structures, memoization where appropriate
4. **Readability** — clear variable names, comments on non-obvious logic, small functions
```

### Project Conventions Skill

```markdown
---
description: Project-specific conventions and patterns
---

## Conventions

- All files use TypeScript strict mode
- Database queries go in `src/db/` — never inline SQL in route handlers
- API responses follow `{ ok: boolean, data?: T, error?: string }` format
- Tests live next to source files: `Button.tsx` → `Button.test.tsx`
- Imports use path aliases: `@/db`, `@/routes`, `@/utils`
```
