Generate images with AI and analyze images using vision models. Enable with `enableImage: true` in agent config.

<Aside>
Maximum image file size for analysis: **20 MB** (`MAX_IMAGE_SIZE`). Default timeout: **120 seconds**. Zero vendor SDK dependencies — all provider APIs are called via direct `fetch`.
</Aside>

**Providers:**
- **Generation** — OpenAI gpt-image-1 / DALL-E (default), Replicate Flux
- **Vision analysis** — OpenAI GPT-4.1-mini (default), Anthropic Claude

**Required environment variables** (depending on provider): `OPENAI_API_KEY`, `REPLICATE_API_TOKEN`, or `ANTHROPIC_API_KEY`.

---

## `image_generate` — Generate Image

Generate an image from a text prompt using an AI model.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `prompt` | string | yes | Text description of the image to generate |
| `path` | string | yes | Destination file path for the generated image |
| `provider` | enum: `openai` \| `replicate` | no | Generation provider (default `openai`) |
| `model` | string | no | Model name (default `gpt-image-1`) |
| `size` | string | no | Image dimensions (default `1024x1024`) |
| `quality` | enum: `auto` \| `high` \| `medium` \| `low` | no | Image quality (OpenAI only) |
| `style` | enum: `vivid` \| `natural` | no | Image style (`dall-e-3` only) |
| `n` | number | no | Number of images to generate (default `1`) |

### Returns

Confirmation with the output file path(s) and image dimensions.

### Notes

- **OpenAI** uses synchronous generation and returns images inline
- **Replicate** uses asynchronous polling with a **2-second interval** and a **4-minute timeout**
- `quality` is only available for OpenAI models
- `style` is only available for the `dall-e-3` model
- When `n > 1`, files are saved with numeric suffixes (e.g. `image_1.png`, `image_2.png`)

## `image_analyze` — Analyze Image

Analyze an image using a vision-language model.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | yes | Path to the image file |
| `prompt` | string | no | Analysis prompt (default `"Describe this image in detail"`) |
| `provider` | enum: `openai` \| `anthropic` | no | Vision provider (default `openai`) |
| `model` | string | no | Model name. OpenAI: `gpt-4.1-mini` (default). Anthropic: `claude-sonnet-4-20250514` (default) |
| `max_tokens` | number | no | Maximum tokens in the response (default `1024`) |

### Returns

Text description or analysis of the image based on the prompt.

### Notes

- Images are **base64-encoded** and sent inline to the vision API
- **Anthropic** supports only: jpeg, png, gif, webp
- **OpenAI** supports a broader range of image formats
- Use a specific `prompt` to guide the analysis (e.g. `"Extract all text from this image"`, `"Describe the UI layout"`)
