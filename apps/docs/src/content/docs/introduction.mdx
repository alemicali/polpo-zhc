## The Problem

AI agents are great at writing code. They're terrible at finishing complex tasks and projects. One agent hallucinates, another breaks what the first one built, a third silently hangs. You end up babysitting the machines that were supposed to save you time — 4 monitors, 12 terminals open, and zero confidence that anything actually works.

**Polpo fixes this.** It's an open-source orchestration framework that makes AI agents reliable by running them as a coordinated team — with plans, scoring, retries, and escalation. You define the work, Polpo makes sure it actually gets done.

Think of it as a project manager with eight arms and infinite patience.

## Why Polpo?

Most agent frameworks let you run *one* agent and hope for the best. Polpo runs a whole team — and keeps them honest:

- **Parallel execution** — Multiple agents work on different tasks at the same time
- **Dependency graphs** — Task B waits for Task A, automatically
- **Quality scoring** — Every task output is scored by an LLM judge (G-Eval). Bad work gets sent back
- **Approval gates** — Need a human to sign off before deploying? Polpo waits. Timeout? Auto-rejects
- **Notifications** — Slack, email, webhook, desktop. Know when things finish, break, or need your attention
- **Escalation** — Agent keeps failing? Polpo swaps it for a better model, reassigns, or pings you
- **Scheduling** — Run plans on a cron. Nightly builds, weekly reports, whatever — Polpo handles the clock
- **Crash recovery** — Agents run as detached processes. Kill the orchestrator, restart it, nothing is lost
- **Zero lock-in** — Swap agents, models, or providers without changing your plans

## Quick Start

<Steps>
  1. **Clone and build**

     ```bash
     git clone https://github.com/openpolpo/openpolpo.git
     cd openpolpo && npm install && pnpm run build
     ```

  2. **Create a plan** (or let the TUI generate one)

     ```json
     {
       "group": "setup-api",
       "tasks": [
         {
           "title": "Create REST endpoints",
           "agent": "backend-dev",
           "description": "Build CRUD API for users and posts"
         }
       ]
     }
     ```

  3. **Run it**

     <Tabs>
       <TabItem label="TUI (interactive)">
         ```bash
         polpo
         ```
       </TabItem>
       <TabItem label="CLI (headless)">
         ```bash
         polpo run
         ```
       </TabItem>
       <TabItem label="HTTP Server">
         ```bash
         polpo serve --port 3890
         ```
       </TabItem>
     </Tabs>
</Steps>

  <Card title="Crash-Resilient" icon="rocket">
    Detached runner processes with file-based state. Survives crashes, reconnects to live agents.
  </Card>

  <Card title="Notifications" icon="email">
    Slack, email, webhook, and desktop notifications for task events, SLA violations, and approvals.

  <Card title="Quality & SLA" icon="star">
