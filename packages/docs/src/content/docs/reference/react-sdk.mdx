The `@openpolpo/react-sdk` package provides type-safe React hooks with real-time SSE updates.

## Installation

```bash
npm install @openpolpo/react-sdk
```

Peer dependencies: React 18 or 19.

## OrchestraProvider

Wrap your app to provide Polpo context:

```tsx
import { OrchestraProvider } from "@openpolpo/react-sdk";

function App() {
  return (
    <OrchestraProvider
      baseURL="http://localhost:3890"
      projectId="my-project"
      apiKey="optional-key"
    >
      <MyApp />
    </OrchestraProvider>
  );
}
```

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `baseURL` | string | yes | Polpo server URL |
| `projectId` | string | yes | Project ID |
| `apiKey` | string | no | API key for authentication |
| `children` | ReactNode | yes | Child components |

## useOrchestra

Access the Polpo client and connection status.

```tsx
const { client, connectionStatus } = useOrchestra();
```

### Returns

| Field | Type | Description |
|-------|------|-------------|
| `client` | `OrchestraClient` | HTTP client with request deduplication |
| `connectionStatus` | `"connected" \| "connecting" \| "disconnected"` | SSE connection state |

## useTasks

List tasks with real-time updates.

```tsx
const { tasks, isLoading, error, createTask, deleteTask, retryTask, refetch } = useTasks();
```

### Returns

| Field | Type | Description |
|-------|------|-------------|
| `tasks` | `Task[]` | All tasks (auto-updated via SSE) |
| `isLoading` | `boolean` | Initial load in progress |
| `error` | `Error \| null` | Fetch error |
| `createTask` | `(opts) => Promise<Task>` | Create a new task |
| `deleteTask` | `(id) => Promise<void>` | Delete a task |
| `retryTask` | `(id) => Promise<void>` | Retry a failed task |
| `refetch` | `() => void` | Force refetch |

## useTask

Single task details with actions.

```tsx
const { task, isLoading, error, updateTask, deleteTask, killTask, reassessTask, retryTask } = useTask("task-1");
```

### Parameters

| Param | Type | Description |
|-------|------|-------------|
| `taskId` | string | Task ID to subscribe to |

### Returns

| Field | Type | Description |
|-------|------|-------------|
| `task` | `Task \| null` | Task object |
| `updateTask` | `(opts) => Promise<Task>` | Update description, assignment, expectations |
| `killTask` | `() => Promise<void>` | Kill running agent |
| `reassessTask` | `() => Promise<void>` | Re-run assessment |
| `retryTask` | `() => Promise<void>` | Retry failed task |

## usePlans

List plans with management actions.

```tsx
const { plans, isLoading, createPlan, executePlan, resumePlan, abortPlan, deletePlan } = usePlans();
```

### Returns

| Field | Type | Description |
|-------|------|-------------|
| `plans` | `Plan[]` | All plans |
| `createPlan` | `(opts) => Promise<Plan>` | Save new plan |
| `executePlan` | `(id) => Promise<{ tasks: Task[] }>` | Execute a draft plan |
| `resumePlan` | `(id, opts?) => Promise` | Resume failed plan |
| `abortPlan` | `(id) => Promise<{ aborted: number }>` | Abort active plan |
| `deletePlan` | `(id) => Promise<void>` | Delete plan |

## useProcesses

Active agent processes.

```tsx
const { processes, isLoading } = useProcesses();
```

### Returns

| Field | Type | Description |
|-------|------|-------------|
| `processes` | `AgentProcess[]` | Active processes with PID, activity, alive status |

`AgentProcess` fields:
- `agentName`, `pid`, `taskId`, `startedAt`, `alive`
- `activity`: `{ lastTool, lastFile, lastUpdate, toolCalls, filesCreated, filesEdited, summary }`

## useEvents

SSE event stream.

```tsx
const { events } = useEvents(["task:*", "assessment:*"], 100);
```

### Parameters

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `filter` | string[] | — | Glob patterns to filter events |
| `maxEvents` | number | `200` | Max events to keep in buffer |

### Returns

| Field | Type | Description |
|-------|------|-------------|
| `events` | `Event[]` | Buffered events (newest first) |

## Architecture

The SDK uses `useSyncExternalStore` for push-based state management:

1. **OrchestraClient** — HTTP client with automatic request deduplication (internal class name)
2. **EventSourceManager** — SSE with auto-reconnect and exponential backoff
3. **OrchestraStore** — Central state with `reduceEvent()` for 35+ event types (internal class name)
4. **Memoized selectors** — WeakMap-cached selectors for each hook
5. **Microtask batching** — `queueMicrotask` for batched re-renders

<Aside>
The SDK has zero runtime dependencies (React is a peer dependency) and is tree-shakeable with `sideEffects: false`.
</Aside>
