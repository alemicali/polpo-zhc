<Warning>
**Deprecated**: These tools have been removed as built-in tools. Use `bash` commands with your package manager directly (e.g. `npm install`, `pnpm add`, `yarn audit`) or install a skill that provides this functionality. The tools are documented here for reference only.
</Warning>

Package management tools that auto-detect your package manager from lockfiles.

<Note>
Supports **npm**, **pnpm**, **yarn**, and **bun**. Auto-detects the package manager by inspecting lockfiles in the project root. Default command timeout is **120 seconds**. All commands run with `bashSafeEnv()`.
</Note>

---

## `dep_install` — Install Dependencies

Run a full dependency install.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `frozen` | boolean | no | Use frozen lockfile mode (`npm ci`, `pnpm --frozen-lockfile`, etc.) |

### Returns

Command output from the package manager install.

### Notes

- When `frozen` is `true`, uses the CI-optimized install variant (`npm ci`, `pnpm install --frozen-lockfile`, `yarn --frozen-lockfile`, `bun install --frozen-lockfile`)
- Default timeout: **120 seconds**

---

## `dep_add` — Add Packages

Add one or more packages to the project.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `packages` | string[] | yes | Package names to add (min 1) |
| `dev` | boolean | no | Install as dev dependencies |
| `exact` | boolean | no | Pin exact versions |

### Returns

Command output confirming the packages were added.

### Notes

- Uses the appropriate `add` / `install` command for the detected package manager
- Default timeout: **120 seconds**

---

## `dep_remove` — Remove Packages

Remove one or more packages from the project.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `packages` | string[] | yes | Package names to remove (min 1) |

### Returns

Command output confirming the packages were removed.

### Notes

- Uses the appropriate `remove` / `uninstall` command for the detected package manager
- Default timeout: **120 seconds**

---

## `dep_outdated` — Check Outdated Packages

List packages that have newer versions available.

### Parameters

No parameters.

### Returns

Table or JSON listing outdated packages with current, wanted, and latest versions.

### Notes

- Timeout: **30 seconds**
- Handles npm exit code 1 gracefully (npm exits with code 1 when outdated packages exist, which is not an error)

---

## `dep_audit` — Security Audit

Run a security audit on project dependencies.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `fix` | boolean | no | Automatically fix vulnerabilities where possible |

### Returns

Audit report listing vulnerabilities by severity.

### Notes

- Timeout: **60 seconds**
- Yarn does **not** support `audit fix` — the `fix` parameter is ignored for Yarn projects

---

## `dep_info` — Package Info

Retrieve registry metadata for a package.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `package` | string | yes | Package name to look up |

### Returns

JSON metadata including version, description, dependencies, and more.

### Notes

- Timeout: **15 seconds**
- Always uses `npm view --json` regardless of the detected package manager
