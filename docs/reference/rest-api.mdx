Polpo exposes a REST API when running in server mode (`polpo serve`). All endpoints are documented with OpenAPI 3.1 and include an interactive playground.

## Chat Completions

The primary way to talk to Polpo programmatically — an OpenAI-compatible endpoint that accepts standard messages and runs Polpo's full 37-tool agentic loop internally. Supports streaming (SSE) and non-streaming.

```bash
curl http://localhost:3000/v1/chat/completions \
  -H "Authorization: Bearer my-secret" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [{"role": "user", "content": "Create a mission to build a REST API"}],
    "stream": true
  }'
```

## Resource Endpoints

All resource endpoints live under `/api/v1/` and cover tasks, missions, agents, sessions, templates, approvals, notifications, skills, config, auth, vault, schedules, memory, files, and logs.

Auth uses `X-API-Key` header or `?apiKey=` query param. Browse the individual endpoints in the sidebar.

## Auth Status

Per-provider authentication health: config keys, environment variables, and OAuth profiles (metadata only — tokens are never exposed).

```bash
curl http://localhost:3000/api/v1/auth/status \
  -H "X-API-Key: my-secret"
```

Returns a `providers` map with `hasConfigKey`, `hasEnvKey`, `oauthAvailable`, and an array of `profiles` with status (`active`, `cooldown`, `billing_disabled`, `expired`), email, expiry, and cooldown details.

## OpenAPI Spec

The full machine-readable spec is available at runtime:

```
GET /api/v1/openapi.json
```
