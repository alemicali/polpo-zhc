The `@lumea-labs/polpo-react` package provides type-safe React hooks with real-time SSE updates.

## Installation

```bash
npm install @lumea-labs/polpo-react
```

Peer dependencies: React 18 or 19.

## PolpoProvider

Wrap your app to provide Polpo context:

```tsx
import { PolpoProvider } from "@lumea-labs/polpo-react";

function App() {
  return (
    <PolpoProvider
      baseUrl="http://localhost:3000"
      projectId="my-project"
      apiKey="optional-key"
    >
      <MyApp />
    </PolpoProvider>
  );
}
```

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `baseUrl` | string | yes | Polpo server URL |
| `projectId` | string | yes | Project ID |
| `apiKey` | string | no | API key for authentication |
| `children` | ReactNode | yes | Child components |
| `autoConnect` | boolean | no | Auto-connect to SSE on mount (default: `true`) |
| `eventFilter` | string[] | no | Glob patterns to filter SSE events |

## useTasks

List tasks with real-time updates.

```tsx

```

### Returns

| Field | Type | Description |
|-------|------|-------------|
| `tasks` | `Task[]` | All tasks (auto-updated via SSE) |
| `isLoading` | `boolean` | Initial load in progress |
| `error` | `Error \| null` | Fetch error |
| `createTask` | `(opts) => Promise<Task>` | Create a new task |
| `deleteTask` | `(id) => Promise<void>` | Delete a task |
| `retryTask` | `(id) => Promise<void>` | Retry a failed task |
| `refetch` | `() => Promise<void>` | Force refetch |

## Imports

The SDK supports subpath exports for tree-shaking:

```tsx
// Full import
import { PolpoProvider, useTasks, useMissions } from "@lumea-labs/polpo-react";

// Direct client import (no React dependency)
import { PolpoClient } from "@lumea-labs/polpo-react/client";

// Individual hook imports
import { useTasks } from "@lumea-labs/polpo-react/hooks/use-tasks";
```

## Architecture

The SDK uses `useSyncExternalStore` for push-based state management:

1. **PolpoClient** — HTTP client with automatic request deduplication
2. **EventSourceManager** — SSE with auto-reconnect and exponential backoff
3. **PolpoStore** — Central state with `reduceEvent()` for 55+ event types
4. **Memoized selectors** — WeakMap-cached selectors for each hook
5. **Microtask batching** — `queueMicrotask` for batched re-renders

<Note>
The SDK has zero runtime dependencies (React is a peer dependency) and is tree-shakeable with `sideEffects: false`.
</Note>
