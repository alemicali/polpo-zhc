<Steps>

1. **Install**

   <Tabs>
     <Tab title="npm">
       ```bash
       npm install -g openpolpo
       ```
     </Tab>
     <Tab title="pnpm">
       ```bash
       pnpm add -g openpolpo
       ```
     </Tab>
     <Tab title="Docker">
       ```bash
       docker pull ghcr.io/openpolpo/openpolpo:latest
       docker run -it -v $(pwd):/workspace ghcr.io/openpolpo/openpolpo:latest
       ```
     </Tab>
   </Tabs>

2. **Init a project**

   ```bash
   mkdir my-project && cd my-project
   polpo init
   ```

   The [setup wizard](/guides/setup) walks you through 4 steps:
   - **Detect providers** — scans your env for API keys and OAuth profiles
   - **Pick a model** — shows recommended models from your detected providers
   - **Name your team** — creates your first agent with a role

   You need at least one API key (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GEMINI_API_KEY`, etc.) or an OAuth login (`polpo auth login`). See [Providers & Models](/models/overview) for 22+ supported providers.

3. **Launch**

   ```bash
   polpo
   ```

   You're in the interactive TUI. Type what you need — "create a REST API", "fix the bug in auth.ts", "write tests for the user module" — hit Enter, and the agent does it.

</Steps>

That's it. One agent, one command, you're live.

## Scale up

When you're ready for more, Polpo orchestrates full teams of agents working in parallel with dependencies and automated quality checks:

- **[Build Your Team](/guides/build-your-team)** — Add specialist agents, configure roles and models
- **[Your First Plan](/guides/first-plan)** — Create multi-task plans with dependencies and automated assessment
- **[Talk from Anywhere](/usage/telegram)** — Control Polpo from Telegram, Slack, or email

## What happens under the hood

1. **Plan** — AI generates tasks with dependencies and agent assignments
2. **Assign** — The orchestrator assigns each task to the right agent, respecting dependencies
3. **Execute** — Detached runner processes spawn agents with coding tools (read, write, edit, bash, glob, grep, ls)
4. **Assess** — Three LLM judges score each result on correctness, completeness, code quality, and edge cases
5. **Done** — Tasks that pass move to `done`. Tasks that fail get feedback and retry automatically
